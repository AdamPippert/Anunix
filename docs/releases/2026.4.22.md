# Anunix 2026.4.22 Release Notes

## Highlights

This release delivers the JEPA world model subsystem, a fully parallel workflow executor
with suspend/resume, the Aether design language, and the iterative belief-action loop
that forms the foundation of Anunix self-improvement.

---

## New Subsystems

### JEPA World Model (`kernel/core/jepa/`)

Joint Embedding Predictive Architecture for continuous system-state learning.
JEPA observes scheduler, memory, routing, and compute counters, encodes them into
a latent vector, and predicts future states to inform routing decisions.

- **Four built-in world profiles**: `anx:world/os-default`, `anx:world/cellular`,
  `anx:world/robotics`, `anx:world/enterprise-it`. Platform code may register
  custom profiles before `anx_jepa_init()`.
- **Encoder / predictor**: transformer-style MLP (256-dim latent, 4-layer encoder)
  sized for AMD XDNA NPU inference (50 INT8 TOPS) with CPU and GPU fallbacks.
- **VICReg training**: variance-invariance-covariance regularization on world model
  updates. Training triggered from workflow execution traces.
- **Route planner hook**: `anx_jepa_route_score_delta()` adds a bounded ±delta to
  engine scoring so JEPA nudges but never overrides a dominant symbolic preference.
- **RLM integration**: JEPA latent state feeds the reinforcement learning module for
  online policy improvement.
- **Data management APIs**: `anx_jepa_world_ingest()`, `anx_jepa_world_rebuild()`,
  `anx_jepa_world_activate()` for dataset ingestion and checkpoint management.

### Iterative Belief-Action Loop (`kernel/core/loop/`, RFC-0020)

Continuous self-improvement loop that drives Anunix toward adapting itself for its
deployment context. Each session records an energy score; the loop detects convergence
and triggers world model rebuilds when improvement stalls.

## Major Enhancements

### Workflow Executor — Phase 2 Parallel DAG Execution

The workflow executor is now a full parallel Kahn's-algorithm DAG runner.

- **Hardware-derived concurrency cap**: `cpu_count/2 + 2` per GPU/NPU accelerator,
  clamped to `ANX_WF_MAX_NODES` and `policy.max_parallel` if set. Derived at create
  time via `anx_wf_cap_compute()`.
- **Suspend-and-escalate failure recovery**: when a node fails, the workflow suspends
  to a first-class `anx_wf_continuation` object (heap-allocated ~8.7 KB) preserving
  full in-degree, completion bitmap, and port-OID binding table. Callers resume with
  `anx_wf_resume(RETRY | SKIP | ABORT | REPLACE)`.
- **By-reference data flow**: all inter-node data passes as `anx_oid_t` — zero-copy,
  live state objects visible to observers throughout execution.
- **Execution traces as state objects**: each node dispatch writes a
  `anx_wf_trace_entry`; `anx_wf_trace_seal()` assembles them into an
  `ANX_OBJ_EXECUTION_TRACE` state object admitted to the L3 memory tier.
- **Recursive sub-workflow nodes**: a `ANX_WF_NODE_SUBFLOW` node dispatches
  `anx_wf_run()` recursively and promotes sub-workflow output OIDs to parent ports.
- **Fan-out / fan-in nodes**: `ANX_WF_NODE_FAN_OUT` broadcasts one OID to N output
  ports; `ANX_WF_NODE_FAN_IN` collects N inputs and gates on all-non-null.

## Design Language

### Aether — Pretty Mode GUI

- New visual design language for `ANX_THEME_PRETTY` mode.
- **ANX Schoolbook 12×24**: custom kernel font replacing IBM VGA 8×16. Correct OTF
  winding direction, full name records, flush and bidirectional serifs, fixed baseline
  alignment for ascenders and descenders.
- **Performance**: per-scanline scroll loop replaced with a single `memmove`; per-pixel
  `putpixel` calls eliminated from font rendering hot path.

## Build System

- JEPA and loop subsystems compile with SSE2 hardware float (pattern rules omit
  `-mgeneral-regs-only`). All other kernel code retains the flag — accidental FPU
  use outside designated ML paths is a compile error.
- **30/30 host-native tests pass.**
