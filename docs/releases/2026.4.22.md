# Anunix 2026.4.22 Release Notes

## Highlights

This release delivers the JEPA world model subsystem, a fully parallel workflow executor
with suspend/resume, the Aether design language, and the iterative belief-action loop
that forms the foundation of Anunix self-improvement.

---

## New Subsystems

### JEPA World Model (`kernel/core/jepa/`)

Joint Embedding Predictive Architecture for continuous system-state learning.
JEPA observes scheduler, memory, routing, and compute counters, encodes them into
a latent vector, and predicts future states to inform routing decisions.

- **Four built-in world profiles**: `anx:world/os-default`, `anx:world/cellular`,
  `anx:world/robotics`, `anx:world/enterprise-it`. Platform code may register
  custom profiles before `anx_jepa_init()`.
- **Encoder / predictor**: transformer-style MLP (256-dim latent, 4-layer encoder)
  sized for AMD XDNA NPU inference (50 INT8 TOPS) with CPU and GPU fallbacks.
- **VICReg training**: variance-invariance-covariance regularization on world model
  updates. Training triggered from workflow execution traces.
- **Route planner hook**: `anx_jepa_route_score_delta()` adds a bounded ±delta to
  engine scoring so JEPA nudges but never overrides a dominant symbolic preference.
- **RLM integration**: JEPA latent state feeds the reinforcement learning module for
  online policy improvement.
- **Data management APIs**: `anx_jepa_world_ingest()`, `anx_jepa_world_rebuild()`,
  `anx_jepa_world_activate()` for dataset ingestion and checkpoint management.

### Iterative Belief-Action Loop (`kernel/core/loop/`, RFC-0020)

Continuous self-improvement loop that drives Anunix toward adapting itself for its
deployment context. Each session records an energy score; the loop detects convergence
and triggers world model rebuilds when improvement stalls.

## Major Enhancements

### Workflow Executor — Phase 2 Parallel DAG Execution

The workflow executor is now a full parallel Kahn's-algorithm DAG runner.

- **Hardware-derived concurrency cap**: `cpu_count/2 + 2` per GPU/NPU accelerator,
  clamped to `ANX_WF_MAX_NODES` and `policy.max_parallel` if set. Derived at create
  time via `anx_wf_cap_compute()`.
- **Suspend-and-escalate failure recovery**: when a node fails, the workflow suspends
  to a first-class `anx_wf_continuation` object (heap-allocated ~8.7 KB) preserving
  full in-degree, completion bitmap, and port-OID binding table. Callers resume with
  `anx_wf_resume(RETRY | SKIP | ABORT | REPLACE)`.
- **By-reference data flow**: all inter-node data passes as `anx_oid_t` — zero-copy,
  live state objects visible to observers throughout execution.
- **Execution traces as state objects**: each node dispatch writes a
  `anx_wf_trace_entry`; `anx_wf_trace_seal()` assembles them into an
  `ANX_OBJ_EXECUTION_TRACE` state object admitted to the L3 memory tier.
- **Recursive sub-workflow nodes**: a `ANX_WF_NODE_SUBFLOW` node dispatches
  `anx_wf_run()` recursively and promotes sub-workflow output OIDs to parent ports.
- **Fan-out / fan-in nodes**: `ANX_WF_NODE_FAN_OUT` broadcasts one OID to N output
  ports; `ANX_WF_NODE_FAN_IN` collects N inputs and gates on all-non-null.

## Design Language

### Aether — Pretty Mode GUI

- New visual design language for `ANX_THEME_PRETTY` mode.
- **ANX Schoolbook 12×24**: custom kernel font replacing IBM VGA 8×16. Correct OTF
  winding direction, full name records, flush and bidirectional serifs, fixed baseline
  alignment for ascenders and descenders.
- **Performance**: per-scanline scroll loop replaced with a single `memmove`; per-pixel
  `putpixel` calls eliminated from font rendering hot path.

## Window Manager and Desktop Applications

### Window Manager (`kernel/core/wm/`)

Full desktop session manager driven by the interface plane compositor.

- **9 virtual workspaces**: switch with Meta+1–9; switching minimizes old-workspace surfaces and restores new ones with keyboard focus intact.
- **Window lifecycle**: `anx_wm_window_open/close/focus/minimize/fullscreen_toggle` — all statically allocated, no heap use in the WM core.
- **Focus cycling**: Meta+Tab advances to the next VISIBLE surface on the active workspace.
- **Pointer routing**: `wm_handle_pointer()` dispatches clicks to the surface under the pointer (z-order-aware); workspace dot hit-test on the menu bar.
- **Fullscreen toggle**: saves and restores pre-fullscreen bounds per-surface.

### Global Hotkey System (`kernel/core/wm/wm_hotkey.c`)

Registry of up to 64 bindings, dispatched before events reach the focused surface.

| Binding | Action |
|---------|--------|
| Meta+1–9 | Switch workspace |
| Meta+Q | Close focused window |
| Meta+F | Fullscreen toggle |
| Meta+Tab | Cycle window focus |
| Meta+Enter | Open shell window (Phase 2) |
| Meta+Space | Command search overlay |
| Meta+W | Workflow designer |
| Meta+O | Object viewer |

`ANX_MOD_META` maps to Super on x86_64 and Cmd on Apple Silicon. Hotkeys fire via a hook in `post_key_event()` before the event is posted to the surface queue.

### Menu Bar (`kernel/core/wm/wm_menubar.c`)

Full-width canvas surface pinned at y=0. Redrawn on workspace switch, window open/close, and every 60 event-loop iterations (≈ clock tick).

- Workspace dots: teal filled = active, dim = inactive.
- Network status dot: teal = up, red = down.
- Power icon: two-rectangle symbol at right edge (shutdown menu Phase 2).
- Teal 2px accent line at bottom edge.
- Raises itself above any newly-focused surface.

### Workflow Designer (`kernel/core/apps/workflow_designer.c`)

Keyboard-driven visual workflow editor (like a Marimo notebook but for programs).

- Views workflow nodes as labeled boxes with teal border on selection.
- Tab/Shift-Tab: cycle node selection. Arrow keys: pan. Meta+W: serialize to DSL. Meta+R: run. Meta+N: add node (cycles kinds). Meta+E: enter edge-add mode. Escape: close.
- Opens the first registered workflow; focuses existing window if already open.

### Object Viewer (`kernel/core/apps/object_viewer.c`)

Live object store browser — the file manager equivalent in an Anunix environment.

- Iterates all live `anx_state_object` entries via `anx_objstore_iterate`.
- Columns: OID (hex), Type, Size. Scrollable row list with selection highlight and detail pane.
- Up/Down: navigate. R: refresh. D: delete (guards on refcount > 1). Enter: log full detail. Escape: close.

### HTTP API — Workflow, Agent, JEPA Endpoints (`kernel/drivers/net/httpd.c`)

New REST endpoints available over port 8080:

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/v1/workflow` | List all registered workflows with state |
| POST | `/api/v1/workflow` | Create workflow from JSON descriptor |
| GET | `/api/v1/workflow/<name>` | Status of named workflow |
| POST | `/api/v1/workflow/<name>/run` | Execute named workflow |
| GET | `/api/v1/workflow/<name>/show` | Emit workflow DSL |
| GET | `/api/v1/workflow/<name>/graph` | ASCII node graph |
| POST | `/api/v1/agent` | Dispatch to agent cell |
| GET | `/api/v1/jepa/status` | JEPA world model status |

### Workflow Serialization (`kernel/core/workflow/workflow_serial.c`)

- `workflow show <name>` — emits editable DSL: `workflow "name" { node N kind "label" { } edge A:P -> B:P }`
- `workflow graph <name>` — BFS depth-bucketed ASCII diagram with `[ kind : label ]` boxes and `|` edge connectors.

### Interface Plane Extensions (`kernel/core/iface/iface.c`)

- `anx_iface_surface_move(surf, x, y)` — reposition a surface.
- `anx_iface_surface_raise(surf)` — move to front of z-list.
- `anx_iface_surface_lower(surf)` — move to back of z-list.
- `anx_iface_surface_at(x, y)` — hit-test: returns topmost VISIBLE surface containing the point.
